Marky 1.0.4

Critical fix: data loss with multiple open documents

  • Fixed: Editing one tab and then switching to another could silently overwrite the first document's in-memory content with the new tab's content. Saving the first document then wrote that content (often empty) to disk.
  • Root cause: a synchronous NSTextView change notification fired through a stale binding during tab switches.
  • Same class of issue in WYSIWYG mode is also fixed — a debounced edit timer could fire after a tab switch and write stale content to the new tab's document.

Cursor preservation in WYSIWYG

  • WYSIWYG mode now remembers each tab's cursor position and restores it when you switch back to that tab. No more flashing cursor at the top-left after a tab switch.
